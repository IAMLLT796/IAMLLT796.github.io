3.用户消息广播机制

基本结构：分为 server部分和client部分

一个客户端就是一个用户，客户端在 server 中通过 user 类进行表示，

上线功能：补充OnlineMap和Message channel，OnlineMap记录当前有哪些用户在线，Message channel 是用来把消息进行广播的，如果当前有一个消息需要让每一个 client 都能够收到，应该通过 Message 来进行广播。将消息发送给 Message 管道中，并且会有一个 goroutine 来监听当前 channel 是否有消息，如果有消息将会去遍历 OnlineMap 来找到每个 user，然后把消息依次发给 user1 的 channel、user2 的 channel…有多少个user对象就发多少次，以此来达到一个广播的效果。

模拟用户上线的功能：当前客户端上线了，客户端就会发送一个 client 请求到服务器，然后对当前的客户端创建一个 user，并将 user 加入到 OnlineMap 中，然后将上线的消息进行广播，将 online 的消息发送到 Message channel 中，监听 Message 的 goroutine 一旦监听到 Message 收到消息就会去 OnlineMap 依次遍历每个 user 并发送消息，当 user 的 goroutine 监听到 channel 收到消息后，就会将消息发送到 客户端。能达到 client1上线的消息通知到 client2 和 client3 的一个广播作用。
