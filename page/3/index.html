2G内存处理4G无序数据

昨天面试遇到这么一个问题：在2G的内存中有4G的数据，要对这些数据进行排序。

不得不说，自己掌握的知识还是太少了，道阻且长！

思路

外部排序+归并排序

  1. 将4G数据分成2个2G数据的文件；
  2. 分别将2个文件写入内存，进行内部排序，输出并存到外部
  3. 使用多路归并排序对2个有序文件取值、排序，输出最小值，并写入外部文件；
  4. 重复第3步，直到生成一个4G的数据。

HashMap的扩容机制

经典面试题之一

如何完美的将扩容过程表达出来及其重要

Java 7 HashMap

  1. HashMap实现了Map接口，即允许放入key为null的元素，也允许插入value为null的元素；
  2. 该容器不保证元素顺序，根据需要可能会对元素重新哈希，元素的顺序也会被重新打散，因此不同时间迭代同一个HashMap的顺序可能不一样；
  3. Java 7采取的是冲突链表方式。

由上图可知，有两个参数影响HashMap的性能：初始容量(initial capacity)和负载系数(load factor)。

  1. 初始容量指定了初始table的大小；
  2. 负载系数指定自动扩容的临界值；
  3. 当entry的数量超过capacity*load_factor时，容器将自动扩容并重新哈希；
  4. 对于插入元素较多的场景，将初始容量设大可以减少重新hash的次数。

将entry对象放入到Hash中，有两个方法可以特别关心：hashcode()和equals()。

  1. hashcode()决定了对象会被放到哪个bucket中；
  2. 当多个对象的哈希值冲突时，equals()决定了这些对象是否是同一个对象

put()

put(key value)方法是将指定的key, value对添加到map里。

该方法首先对map做一次查找，看是否包含该元组，如果已经包含则直接返回。查找的过程类似于getEntry()过程。

如果没找到，则会通过addEntry(hash, key, value, bucketIndex)方法插入新的entry。采用头插法！

Java 8 HashMap

底层实现：数组+链表+红黑树

相对于Java 7 HashMap的查找时间复杂度为O(n)，Java 8 HashMap的查找时间复杂度为O(logn)。

扩容过程

HashMap默认的初始化长度为16.当HashMap中的元素个数之和大于负载因子*当前容量的时候就要进行扩充，容量变为原来的2倍。

当链表长度大于阈值（8）时，如果此时当前数组的长度小于64，那么会先进行数组扩容，而不是转换成红黑树。如果数组大于64，则直接将链表转换成红黑树，以减少搜索时间，并且采用尾插法！

HashMap的put过程对比

  1. 根据key通过哈希算法和与运算得出数组下标；

  2. 如果数组下标为空，则将key和value封装成Entry对象或者Node对象，并放入该位置；

  3. 如果数组下标不为空，则要分情况讨论：

    1. 在JDK1.7中，则会先判断是否需要扩容，如果要扩容就进行扩容，如果不需要扩容就生成Entry对象，并使用头插法添加到当前链表中；

    2. 在JDK1.8中，则会先判断当前位置上的Node类型，看是红黑树节点还是链表节点：

      ①如果是TreeNode，则将key和value封装成一个红黑树结点并添加到红黑树中，在这个过程中会判断红黑树中是否存在当前key，如果存在则更新当前key的value;

      ②如果是链表节点，则将key和value封装成一个Node并通过尾插法插入到链表的最后位置去，因为是尾插法，所以需要遍历链表，在遍历过程中会判断是否存在当前key，如果存在则更新其value，当遍历完后，将新的Node插入到链表中，插入到链表后，会看当前链表的结点个数，如果大于8，则会将链表转为红黑树；

      ③将key和value封装为Node插入到链表和红黑树后，再判断是否需要扩容，如果需要扩容，就结束put方法。

常见返回结果的HTTP状态码

HTTP状态码负责表示客户端HTTP请求的返回结果、标记服务器端的处理是否正常、通知出现的错误等工作。

状态码由3位数字和原因短语组成。

2XX成功

200 OK

表示从客户端发来的请求在服务器端被正常处理了。

204 No Content

表示服务器接收到的请求已经成功处理，但在返回的响应报文中不包含实体的主体部分。

206 Partial Content

表示客户端进行了范围请求，而服务器成功执行了这部分的GET请求。

3XX重定向

301 Moved Permanently

永久性重定向。该状态码表示请求的资源已经被分配了新的URI，以后应该使用资源现在所指的URI。

302 Found

临时性重定向。该状态码表示请求的资源已经被分配了新的URI，希望用户这次能够使用新的URI访问。

与301状态码相似，但是302表示资源不是永久移动，只是临时性质的。

303 See Other

表示请求对应的资源存在着另一个URI，应该使用GET方法定向获取请求的资源

303状态码与302状态码有着相同的功能，但是303状态码表示客户端应当采用GET方法获取资源。

304 Not Modified

表示客户端发送附带信息的请求时，服务器端允许请求访问资源，但是因发生请求未满足条件时，直接返回304 Not Modified。

307 Temporary Redirect

临时重定向。与302有着相同的含义。

4XX客户端错误

400 Bad Request

表示请求报文中存在语法错误。

401 Unauthorized

表示发送的请求需要有通过HTTP认证的认证信息。

403 Forbidden

表示对请求资源的访问被服务器拒绝了。

404 Not Found

表示服务器上无法找到请求的资源。

5XX服务器错误

500 Internal Serval Error

表示服务器在执行过程中发生了错误。

503 Service Unavailable

表示服务器暂时处在超负载或者正在进行停机维护，现在无法处理请求。

SpringBoot中@Autowired报红及解决方案

问题产生

SpringBoot中的service层经常需要将mapper注入进来，但是注入一个mapper接口时经常会报红。

报红原因

@Autowired是Spring的注解，该报红是提示找不到对它的bean，因为程序中没有显示的将userMapper注入到Spring容器中管理。

解决方案1

在UserMapper.java接口上添加@Repository注解即可，该注解是Spring的注解，将当前类注解到Spring容器中实例化为一个bean，所以@Autowired就可以找到这个bean了。

已解决

解决方案2

直接将Autowired换成@Resource注解，@Resource是JDK中的注解，不会像@Autowired那样去Spring容器中去寻找bean。也不需要去接口中添加@Repository注解。

LRU的写法

复习LinkedHashMap时，学习到了LinkedHashMap的经典用法。所以记录一下。

LinkedHashMap经典用法

LinkedHashMap除了可以保证迭代顺序外，还有一个非常有用的用法: 可以轻松实现一个采用了FIFO替换策略的缓存。具体说来，LinkedHashMap有一个子类方法protected boolean removeEldestEntry(Map.Entry<K,V> eldest)，该方法的作用是告诉Map是否要删除“最老”的Entry，所谓最老就是当前Map中最早插入的Entry，如果该方法返回true，最老的那个元素就会被删除。在每次插入新元素的之后LinkedHashMap会自动询问removeEldestEntry()是否要删除最老的元素。这样只需要在子类中重载该方法，当元素个数超过一定数量时让removeEldestEntry()返回true，就能够实现一个固定大小的FIFO策略的缓存。

位运算知识点总结

主要的位运算概览

符号 描述 运算规则
& 与 两个位都为1时，结果才为1
| 或 两个位都为0时，结果才为0
^ 异或 两个位相同为0，相异为1

按位与运算符号 (&)

定义

参加运算的两个数据，按照二进制位运算进行与运算

运算规则

即两位同时为1，结果才为1，否则结果为0

例子

与运算用途

清零

如果想将一个数清零，即将其二进制位全为0，只要与一个各位都为0的数值相与，结果就为0。

取一个数的指定位

比如要取数X=10101110的低四位，只需要找一个数Y，令Y的低四位为1，其余位为0，即Y=0000 1111。然后将X与Y进行与运算就可以得到X的低四位了。

判断奇偶

只要根据最末尾是0还是1来决定，为0就为偶数，为1就是奇数。

按位或运算符（|）

定义

参加运算的两个对象，按照二进制进行或运算

运算规则

即参加运算的两个对象只要有一个为1，其结果就为1.

或运算用途

对一个数据的某些位设置为1

比如将X=1010 1110的低四位设置为1，只需要另一个数Y=0000 1111。然后就可以将X与Y进行按位或运算。

异或运算符（^）

定义

参与运算的两个数据，按二进制位进行异或运算

运算规则

即参加运算的两个对象，如果两个相应位相同为0，相异为1

异或的性质

  1. 交换律
  2. 结合律：(a^b)^c=a^(b^c)
  3. 对于任何数x，都有x^0=x，x^x=0
  4. 自反性：a^b^b=a^0=a

翻转指定位

比如将数X=1010 1110的低四位进行翻转，只需要令Y的低四位为1，其余位为0，即Y=0000 1111。
